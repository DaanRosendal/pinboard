# AGENTS.md — Pinboard VS Code Extension

## What this is

A VS Code extension (TypeScript + esbuild) that adds a Pinboard panel to the Explorer sidebar. Users pin arbitrary disk files and folders as persistent shortcuts, independent of the workspace.

## Stack

- **Language**: TypeScript (strict mode)
- **Bundler**: esbuild via `esbuild.js`
- **Runtime target**: VS Code Extension Host (Node.js, CommonJS)
- **Min VS Code version**: 1.85.0
- **No runtime dependencies** — devDependencies only

## Key commands

```bash
npm run compile      # type-check (tsc --noEmit) + lint (eslint) + bundle (esbuild)
npm run watch        # parallel: esbuild --watch + tsc --noEmit --watch
npm run package      # production bundle (minified, no sourcemaps)
```

There are no automated tests. Manual testing is done via F5 in VS Code (launches Extension Development Host).

## Project structure

```
src/
  extension.ts         # activate() / deactivate() — wires commands + treeView
  PinboardProvider.ts  # all logic: TreeDataProvider, DragAndDropController, commands
dist/
  extension.js         # compiled output (not committed, generated by esbuild)
esbuild.js             # bundler config
eslint.config.js       # ESLint 9 flat config
```

Everything lives in two source files. Keep it that way unless there's a strong reason to split.

## Architecture constraints

- **State**: `globalState` (global scope) or `workspaceState` (workspace scope) via `ExtensionContext`. Key: `pinboard.paths`. Never use the filesystem to persist state.
- **UI**: `TreeView` only — no WebviewView. The view ID `"pinboard"` must match in `package.json` and `createTreeView()`.
- **Tree item types**: `PinnedItemRoot` (pinned roots, `kind: 'root'`) and `FileSystemItem` (nested files/dirs, `kind: 'fsitem'`). Both extend `vscode.TreeItem`.
- **contextValue strings** (must stay in sync with `package.json` `when` clauses):
  - `pinnedFolder` — root folder, not current workspace
  - `pinnedFolderActive` — root folder, is current workspace
  - `pinnedFileRoot` — root file (pinned directly)
  - `pinnedDirectory` — nested directory inside a pinned folder
  - `pinnedFile` — nested file inside a pinned folder
- **Context key**: `pinboard.isGlobalScope` (boolean) — set via `setContext`, drives scope toggle button visibility in `view/title`.

## package.json rules

- `contributes.views.explorer[].id` must equal the first arg of `createTreeView()`
- `"group": "inline"` puts buttons directly on tree rows on hover; all other groups go in right-click menu
- Command icons use VS Code codicons: `$(icon-name)` syntax
- `activationEvents: ["onView:pinboard"]` — lazy activation, do not change to `*`

## Code style

- No classes beyond the two existing TreeItem subclasses and the provider
- Async methods return `Promise<void>`; synchronous methods return void or a value directly
- Private helpers at the bottom of the class, public command methods in the middle
- No comments unless logic is non-obvious
